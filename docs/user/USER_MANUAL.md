# Nolia 使用手册

Nolia 是一款 macOS 本地优先 Markdown 知识工作台。用户正文保存在自己选择的工作区目录中；`.nolia/` 目录只保存索引、快照、缓存和应用元数据。

## 快速开始

1. 安装 `Nolia.app`，首次打开如遇 macOS 安全提示，可在「系统设置 -> 隐私与安全性」中允许打开。
2. 在首页选择「打开工作区」或「创建工作区」，工作区就是 Nolia 读写文件的根目录。
3. 在文件树中新建 Markdown 文档，或打开已有 `.md` / `.markdown` 文件。
4. 使用顶部工具栏在「编辑」「Markdown 源码」「分屏预览」之间切换。
5. 通过活动栏查看最近、笔记、收藏、搜索和反向链接。

## 安装与数据

- 支持 macOS 13 及以上版本。
- 正式发布包应优先选择与本机架构匹配的 DMG；通用包体积更大。
- 升级时直接替换 `/Applications/Nolia.app`，不会主动迁移或删除工作区文件。
- 卸载应用不会删除工作区；如需清理全局状态，可删除 `~/Library/Application Support/Nolia/`。
- 推荐把工作区纳入 Time Machine、Git 或云盘备份，但同步目录可能产生外部修改冲突。

## 主界面

- 活动栏：切换最近、笔记、收藏、搜索等视图。
- 侧边栏：展示当前视图列表、搜索结果或文件树。
- 编辑区：编辑 Markdown、JSON、TXT，或预览图片、PDF、音视频等资源。
- 右侧面板：显示目录、反向链接等辅助信息。
- 状态栏：显示保存状态、字数、行数和当前编辑模式。

## 文件与格式

文件树支持新建、打开、重命名、删除、拖拽移动、文件夹展开收起和右键菜单。

内置处理规则：

- Markdown：`.md`、`.markdown`。
- JSON：`.json`，支持校验、格式化、压缩和排序键。
- 文本：`.txt`、`.log`、`.csv`、`.yaml`、`.yml`、`.toml`、`.xml`、`.html`、`.js`、`.ts`、`.tsx` 等。
- 资源预览：图片、PDF、音频、视频、压缩包和未知二进制文件。
- 插件可扩展新的文件编辑器或预览器。

## Markdown 编辑

编辑模式提供所见即所得体验，源码模式使用 CodeMirror，分屏模式并排显示源码和预览。

常用语法：

```md
# 标题
**加粗** *斜体* ~~删除线~~ `行内代码`
- 无序列表
1. 有序列表
- [ ] 任务项
> 引用
[链接](https://example.com)
![图片](assets/example.png)
```

支持 GFM 表格、脚注、数学公式、Mermaid、HTML 块、代码块和扩展块。复杂语法在编辑模式下会尽量显示为渲染结果；选中或点击后可回到原始 Markdown 源码进行修改，失焦后重新渲染。

代码块示例：

````md
```ts
console.log("Nolia");
```
````

公式示例：

```md
$E = mc^2$

$$
\int_0^1 x^2 dx
$$
```

表格支持插入、行列调整、对齐和源码编辑。发布前应使用 `examples/markdown-syntax-test.md` 做人工回归。

## 搜索、收藏与反向链接

- 搜索会匹配 Markdown 标题、正文和路径，支持中文。
- 最近浏览和最近编辑会根据工作区记录恢复。
- 收藏用于固定常用文档。
- 反向链接会显示链接到当前文档的页面和未链接提及。

## 设置与快捷键

设置入口：`Command+,` 或菜单「Nolia -> 设置」。

常用快捷键：

| 快捷键 | 功能 |
| --- | --- |
| `Command+N` | 新建文档 |
| `Command+O` | 打开工作区或文件 |
| `Command+S` | 保存当前文档 |
| `Command+Shift+P` | 命令面板 |
| `Command+F` | 当前编辑器搜索 |
| `Command+B` | 加粗 |
| `Command+I` | 斜体 |
| `Command+Shift+E` | 导出 |

设置项包括主题、编辑模式、编辑区宽度、字体大小、专注模式和插件管理。

## 插件

外部插件目录：

```text
~/Library/Application Support/Nolia/plugins/<pluginId>/
```

插件默认需要在设置中接受权限并手动启用。权限变化后必须重新确认。开发说明见 [插件开发指南](../plugins/PLUGIN_DEVELOPMENT.md)。

## 故障排查

- 应用无法打开：确认 macOS 安全设置、架构匹配和应用完整性。
- 最近工作区无法打开：确认目录仍存在且当前用户有读写权限。
- 文件树不刷新：点击刷新、重启应用，或确认外部同步工具未锁定文件。
- 自动保存异常：检查状态栏提示，避免同时用其他编辑器修改同一文件。
- Markdown 复杂语法无法编辑：切换到源码模式，或选中语法块后编辑原始 Markdown。
- JSON 格式化失败：先根据诊断修复语法错误。
- 插件不显示：确认插件目录、`plugin.json`、权限确认和安全模式状态。
- 资源无法预览：确认文件未损坏，未知格式会回退到通用资源视图。
