# QA 测试计划

本文是 Nolia 发布前测试的唯一维护入口，合并了原全量回归矩阵和桌面验收清单。最终放行仍以 [发布检查清单](../release/RELEASE_CHECKLIST.md) 为准。

## 测试对象

- 自动化对象：当前源码、main/preload/renderer 构建产物和 Playwright 开发模式窗口。
- 桌面对象：当前系统重新打包并安装后的 Nolia；必要时补充验证 `release/*unpacked*`。
- 工作区：至少准备一个临时空工作区、一个包含真实文档副本的大工作区、一个覆盖 Markdown 边界语法的工作区。
- 数据保护：桌面验收前备份 Nolia 全局应用数据目录；真实工作区只用副本测试。

## 自动化基线

发布前必须全部通过：

```sh
npm run typecheck
npm run lint
npm test
npm run e2e
npm run build
```

可先按改动范围跑局部测试，但合入和打包前必须跑全量。

| Markdown 编辑/分屏 | 三种模式切换、预览同步、复杂语法源码编辑、表格行列操作、公式、Mermaid、脚注、HTML |
| 资源编辑 | JSON 校验/格式化/排序/压缩，文本语言识别、搜索、换行、行号、空白符显示和清理 |

## AI 冒烟测试

1. OpenAI-compatible：保存 API key，刷新模型，测试连接，发送“你好”并确认返回文本。
2. Base URL：分别使用裸域名和 `/v1` API root 测试聊天请求，正式运行不能出现测试通过但聊天空响应。
3. Ollama：验证 native 或 OpenAI-compatible 模式至少一种可用。
4. 权限：关闭 workspace read 后不能读取任意工作区文件；关闭写入权限后不能生成 workspace proposal。
5. 写入：当前文档 patch 和 workspace patch 必须等待用户确认，确认前不能落盘。
6. Semantic index：未配置 embedding 时全文检索可用；配置后可创建、更新、清空索引。

## 通过标准

- 自动化基线全部通过。
- 安装版至少完成启动、工作区、文件树、Markdown、资源编辑、设置、AI、菜单和系统集成的高优先级路径。
- 无 P0/P1 问题；P2 问题必须记录影响面、规避方式和处理计划。
- 截图、视频、临时工作区、测试报告、打包产物不提交到 Git。
