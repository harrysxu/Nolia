# Nolia 全量回归测试用例

本文定义 Nolia macOS 桌面版的完整回归测试基线。测试对象优先为重新打包并安装到 `/Applications/Nolia.app` 的应用；开发模式 E2E 只能作为补充验证。

## 1. 测试范围与前置条件

- 测试平台：macOS，本机中文界面，应用名称 `Nolia`。
- 测试工作区：准备一个临时空工作区、一个包含大量文件的真实工作区、一个包含全部 Markdown 语法的文档。
- 数据保护：测试前备份 `~/Library/Application Support/Nolia/global-state.json`；真实工作区只做只读或使用副本。
- 自动化命令：`npm run typecheck`、`npm run lint`、`npm test`、`npm run e2e`、`npm run build`。
- 手工验收：使用安装版 App 操作菜单、窗口、文件选择器、系统打开方式、拖拽和复杂 UI 交互。

## 2. 启动、首页与工作区

## 15. 自动化覆盖建议

- 单元测试覆盖：Markdown 解析/转换、目录生成更新、i18n、workspace tree、settings、workspace db/service、file system service、extensions。
- E2E 覆盖：welcome、full-smoke、markdown-boundary、resource-editors、plugin-rendering、i18n、ui-visual、outline-manual。
- Computer Use 手工覆盖：系统菜单、原生文件对话框、Finder 右键打开、安装版窗口行为、真实大工作区性能、视觉细节。

## 16. 发布前通过标准

- 所有自动化命令通过。
