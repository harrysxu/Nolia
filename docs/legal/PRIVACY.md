# Nolia 隐私声明

生效日期：2026-05-31

本文说明 Nolia 当前版本的数据处理方式。Nolia 是本地优先桌面应用，不提供官方云同步，也不会默认把用户笔记上传到官方服务器。

## 本地优先

Nolia 是本地优先应用。默认情况下：

- 不要求注册账户。
- 不内置官方云同步。
- 不把用户笔记上传到官方服务器。
- 不内置遥测上报。
- 用户文档保存在用户选择的本地工作区。

## 应用会读取和写入哪些数据

应用会访问：

- 用户主动选择的工作区目录。
- 工作区内的 Markdown、文本、JSON、附件和资源文件。
- 工作区 `.nolia` 目录中的索引、缓存、日志和快照。
- 全局应用数据目录中的设置、最近工作区和插件状态。

全局应用数据目录：

```text
macOS:   ~/Library/Application Support/Nolia/
Windows: %APPDATA%\Nolia\
Linux:   ~/.config/Nolia/
```

## 派生数据

为了提供搜索、反向链接、最近记录和资源索引，应用会从 Markdown 文件中生成派生数据，例如：

- 标题。
- 正文纯文本。
- 标签。
- 链接。
- 附件引用。
- 最近打开和编辑记录。

这些数据保存在本机。

## 插件

外部插件可能申请额外权限，例如读取工作区、写入文件、访问剪贴板或发起网络请求。

Nolia 会在启用外部插件前显示权限确认。用户应只安装信任来源的插件。

外部插件造成的数据读取、写入、网络传输或文件修改，取决于插件自身实现和用户授予的权限。

## 网络访问

核心应用默认不需要上传用户笔记。

可能发生网络访问的情况包括：

- 用户打开外部链接。
- 用户安装并启用了具有网络权限的插件。
- 用户启用 AI Assistant，并配置 OpenAI-compatible、OpenAI Responses 或其他云端模型服务。
- 未来版本中用户主动启用的联网能力。

启用云端 AI 服务时，请求可能包含用户输入的提示词、当前文档片段、工作区检索结果、语义索引检索片段、文件名或路径等上下文。请求发送到用户配置的模型服务商或网关，Nolia 不会通过官方服务器中转这些请求。使用本地 Ollama 时，请求默认发送到本机 Ollama 服务。

AI API key 保存在本机。应用会优先使用系统安全存储；如果系统安全存储不可用，会使用本地应用数据目录中的降级存储。

## 诊断信息

Nolia 会在本机写入诊断日志，用于排查启动、窗口、工作区、插件和索引问题。诊断日志不会自动上传。

诊断日志可能包含时间、错误信息、窗口状态、插件 ID、工作区路径、文件路径或系统错误堆栈。日志不应默认记录 Markdown 正文内容。用户可以通过应用中的日志入口查看日志，并自行决定是否分享给维护者。

## 用户控制

用户可以：

- 选择工作区位置。
- 删除工作区中的 `.nolia` 派生数据。
- 删除全局应用数据目录。
- 停用或删除外部插件。
- 通过系统权限控制文件访问。

## 数据删除

卸载应用本体不会自动删除用户工作区。

如需彻底清理：

1. 卸载或删除 Nolia 应用本体。
2. 删除当前系统的 Nolia 全局应用数据目录。
3. 如不再需要笔记，手动删除用户工作区。
