# Nolia 隐私声明

生效日期：2026-05-31

本文说明 Nolia 当前版本的数据处理方式。正式公开发布前，应由发布者按实际分发渠道、支持邮箱和法律辖区复核。

## 本地优先

Nolia 是本地优先应用。默认情况下：

- 不要求注册账户。
- 不内置官方云同步。
- 不把用户笔记上传到官方服务器。
- 不内置遥测上报。
- 用户文档保存在用户选择的本地工作区。

## 应用会读取和写入哪些数据

应用会访问：

- 用户主动选择的工作区目录。
- 工作区内的 Markdown、文本、JSON、附件和资源文件。
- 工作区 `.nolia` 目录中的索引、缓存、日志和快照。
- 全局应用数据目录中的设置、最近工作区和插件状态。

全局应用数据目录：

```text
~/Library/Application Support/Nolia/
```

## 派生数据

为了提供搜索、反向链接、最近记录和资源索引，应用会从 Markdown 文件中生成派生数据，例如：

- 标题。
- 正文纯文本。
- 标签。
- 链接。
- 附件引用。
- 最近打开和编辑记录。

这些数据保存在本机。

## 插件

外部插件可能申请额外权限，例如读取工作区、写入文件、访问剪贴板或发起网络请求。

Nolia 会在启用外部插件前显示权限确认。用户应只安装信任来源的插件。

外部插件造成的数据读取、写入、网络传输或文件修改，取决于插件自身实现和用户授予的权限。

## 网络访问

核心应用默认不需要上传用户笔记。

可能发生网络访问的情况包括：

- 用户打开外部链接。
- 用户安装并启用了具有网络权限的插件。
- 未来版本中用户主动启用的联网能力。

## 诊断信息

当前项目包含诊断能力。诊断信息应只用于排查问题，不应默认包含用户 Markdown 正文。

正式发布前建议明确：

- 是否收集诊断。
- 是否由用户手动导出。
- 诊断是否包含文件路径、系统版本、错误堆栈或插件信息。
- 支持邮箱或反馈渠道。

## 用户控制

用户可以：

- 选择工作区位置。
- 删除工作区中的 `.nolia` 派生数据。
- 删除全局应用数据目录。
- 停用或删除外部插件。
- 通过系统权限控制文件访问。

## 数据删除

卸载应用本体不会自动删除用户工作区。

如需彻底清理：

1. 删除 `/Applications/Nolia.app`。
2. 删除 `~/Library/Application Support/Nolia/`。
3. 如不再需要笔记，手动删除用户工作区。
