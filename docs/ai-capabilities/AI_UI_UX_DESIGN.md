## 1. 设计结论

Nolia AI 的界面不应设计成一个独立聊天应用，而应成为 Nolia 编辑器、工作区搜索、反链、附件和插件系统的一层增强能力。

最终 UI 采用以下结构：

1. 右侧 AI 面板作为主工作区，承载对话、上下文预览、引用来源和结果操作。
2. 命令面板、编辑器工具栏、选区浮动菜单和右键菜单作为快捷入口。
3. 设置弹窗新增 AI 分类，承载 provider、模型、隐私、索引和命令管理。
4. Agent 生成的多文件变更进入专门的变更审核界面，不直接落盘。
5. 所有 AI 请求都展示上下文范围，所有写入都由用户显式确认。

设计目标是让 AI 变得“可见、可控、可撤销、可追溯”，而不是让用户猜测 AI 读了什么、写了什么、发给了谁。

## 2. 竞品参考与设计取舍

### 2.1 Notion AI

参考点：

- AI 可以搜索工作区、连接应用和 Web。
- 设置中提供 AI Web Search 开关，以及外部 Web 请求确认。
- AI 与页面编辑、数据库和工作区搜索深度结合。
- 提供全局快捷入口和底部 AI 入口。

对 Nolia 的启发：

- AI 入口要贴近编辑器和搜索，而不是只放在设置或菜单里。
- 本地工作区和附件来源必须在 UI 中分开标识。
- Nolia 当前版本不提供 Web Search 开关，也不在 UI 中出现联网搜索入口；Notion 的 Web Search 仅作为未来能力参考。
- 不能只给答案，要给来源和跳转能力。

参考链接：

- https://www.notion.com/help/notion-ai-faqs
- https://www.notion.com/help/search

### 2.2 Craft Assistant

参考点：

- Assistant 面板支持文档级和 Space 级使用。
- 模型可以在面板内切换。
- Explore 模式提出变更并等待批准，Execute 模式直接应用。
- 支持会话、多会话、后台完成提示、撤销上一轮 Assistant 动作。
- 强调最小必要数据发送和本地模型。

对 Nolia 的启发：

- Nolia 应保留“提议/审核”作为默认模式，不默认直接执行。
- 模型、范围和模式选择应靠近输入框，而不是藏在深层设置。
- AI 写入后必须能整体撤销，至少要能通过现有历史快照恢复。
- 本地模型需要在 UI 中作为清晰的隐私选项，而不只是技术配置。

参考链接：

- https://support.craft.do/en/ai-assistant/using
- https://support.craft.do/en/ai-assistant/editing
- https://support.craft.do/en/ai-assistant/models

### 2.3 Obsidian Copilot

参考点：

- 围绕 vault 做 Vault QA、当前文件/选区命令和自定义命令。
- 支持命令面板、右键菜单、slash command。
- 支持 embedding、索引刷新、排除文件夹/标签/文件名。
- 支持 OpenAI-compatible 和本地模型。

对 Nolia 的启发：

- AI 命令必须是“预置 + 自定义 + 插件”的统一系统。
- 工作区索引状态和排除规则必须可见。
- 用户需要知道当前命令会作用于选区、当前文件还是整个工作区。
- 命令可以来自 Markdown prompt 文件或工作区配置，但 UI 中必须可管理。

参考链接：

- https://www.obsidiancopilot.com/en/docs/settings
- https://www.obsidiancopilot.com/en/docs/vault-qa
- https://www.obsidiancopilot.com/en/v4

### 2.4 NotebookLM

参考点：

- 强调 source-grounded answers。
- 回答中展示引用，用户可以回到原始来源验证。
- 以来源集合为核心，而不是泛化聊天。

对 Nolia 的启发：

- 工作区问答必须把“来源”放到答案旁边，而不是藏在 debug 信息里。
- 引用点击后应打开文件并跳转到标题或行号。
- 没有可靠来源时，UI 应显示证据不足，而不是给出看似确定的回答。

参考链接：

- https://notebooklm.google/
- https://support.google.com/notebooklm/

### 2.5 Cursor

参考点：

- Agent 生成的变更通过 diff review 展示。
- 用户可以逐文件、逐块接受或拒绝。
- Review changes 是 Agent 工作流的一部分，而不是事后补救。

对 Nolia 的启发：

- Nolia 的审核式 Agent 必须有专门的变更计划界面。
- 文件创建、修改、重命名、删除要分类型展示。
- 删除和移动必须比普通修改更强确认。
- 多文件变更不能只显示一段自然语言总结。

参考链接：

- https://docs.cursor.com/en/agent/review

## 3. 设计原则

### 3.1 Workspace-native

AI 应像搜索、反链、目录一样属于 Nolia 工作台，不是外挂窗口。入口与结果都要能回到当前文件、当前选区、当前工作区。

### 3.2 Context-first

每次请求先让用户看到上下文范围：

- 选区
- 当前段落
- 当前文档
- 当前文件夹
- 工作区搜索结果
- 反向链接
- 附件内容
- Web 来源（未来能力，当前版本不展示）

如果上下文不可见，用户无法判断隐私和结果可信度。

### 3.3 Safe-by-default

默认行为：

- AI 默认关闭。
- 当前版本不支持 Web 搜索，UI 不展示 Web 搜索入口。
- 工作区上下文默认需要确认。
- 写入默认需要确认。
- Agent 默认只提出变更计划。
- 删除、重命名、移动必须单独确认。

### 3.4 Source-grounded

工作区问答的回答必须显示来源。来源不是装饰，而是主要交互对象：可展开、可复制、可跳转、可排除。

### 3.5 Command-first

Nolia 是编辑器和知识工作台，高频 AI 能力应能通过命令面板快速触发。鼠标入口服务发现，键盘入口服务效率。

### 3.6 Reversible

用户必须能撤销或拒绝 AI 修改：

- 单点编辑进入编辑器 undo stack。
- 文件级修改显示 diff。
- 批量变更逐项接受。
- 写入前创建历史快照。

### 3.7 Quiet utility

视觉风格要克制、信息密度适中，服务长时间写作和整理。避免大面积渐变、营销式 hero、拟人化角色和过度动效。

## 5. 入口设计

### 5.1 右侧 AI 面板入口

位置：

- 编辑器顶部栏右侧增加 AI 图标按钮。
- 右侧面板 Header 支持切换到 AI。
- 命令面板提供 `打开 AI 助手`。

图标建议：

- `Sparkles`：AI 助手主入口。
- `Bot`：会话状态。
- `ShieldCheck`：隐私安全状态。
- `DatabaseZap`：AI 索引状态。

行为：

- 点击后打开右侧面板并切换到 `AI`。
- 如果未配置 provider，显示配置引导。
- 如果当前右侧面板宽度小于 420px，首次打开 AI 时建议扩展到 460px，但保留用户手动宽度。

### 5.2 命令面板入口

AI 命令进入现有 CommandPalette：

```text
AI: 总结选区
AI: 润色选区
AI: 翻译为英文
AI: 总结当前文档
AI: 问当前文档
AI: 问工作区
AI: 生成标签
AI: 推荐双链
AI: 管理自定义命令
```

命令显示规则：

- 有选区时，选区命令靠前。
- 无选区但有当前文档时，当前文档命令靠前。
- 无工作区时，工作区命令隐藏或 disabled。
- Provider 未配置时，AI 命令可见但执行后进入设置引导。

### 5.3 编辑器工具栏入口

Source 和 WYSIWYG 工具栏都新增 AI 按钮：

- 图标：`Sparkles`
- 点击打开菜单：
  - 总结
  - 改写
  - 润色
  - 翻译
  - 提取待办
  - 更多命令

该入口服务发现，不替代命令面板。

### 5.4 选区浮动菜单

当用户选中文本时，显示轻量浮动菜单：

```text
[Sparkles] [润色] [总结] [翻译] [更多]
```

规则：

- 菜单不遮挡选中文本关键区域。
- 在代码块、表格、数学公式内仍显示，但命令范围要明确。
- Escape 或点击空白关闭。
- 如果选区过短，只显示 `解释`、`扩写`、`更多`。

### 5.5 右键菜单

右键菜单增加 `AI` 子菜单：

```text
AI
├─ 总结选区
├─ 润色
├─ 翻译为...
├─ 解释这段内容
└─ 运行自定义命令...
```

右键菜单用于低频发现，不承载复杂配置。

## 6. AI 面板布局

### 6.1 默认宽度

建议：

- 默认宽度：460px。
- 最小宽度：360px。
- 推荐宽度：460-560px。
- 最大宽度：720px。
- 小于 400px 时进入 compact mode，隐藏部分 secondary label。

当前项目已有右侧面板宽度持久化，AI 面板应复用该机制。

### 6.2 Header

结构：

```text
AI 助手          [Provider: Ollama] [Model: llama3.2] [设置]
[选区] [当前文档] [文件夹] [工作区]
```

状态：

- 未配置：`未配置模型`
- 本地：`本地 · Ollama`
- 云端：`云端 · OpenAI`
- 插件：`插件 · plugin-name`
- 索引中：`索引 42%`
- Web 搜索：当前版本不展示状态 chip；未来支持时再显示网络访问状态。

点击状态 chip 可打开对应设置或详情。

### 6.3 Context Strip

始终位于输入框上方或面板顶部第二行。

示例：

```text
将发送：选区 328 字符 · 当前文档结构 · 工作区 6 个片段
[查看详情] [调整]
```

chip 规则：

- `选区`：绿色/安全语义，表示用户显式选中。
- `当前文档`：中性。
- `工作区`：强调，需要确认。
- `附件`：强调，需要确认。
- `Web`：未来能力；当前版本不展示 Web chip。
- `插件 provider`：显示插件名。

Context Strip 不应只在首次请求出现。每次请求都要能看到。

### 6.4 Conversation 区

消息类型：

- 用户消息。
- AI 流式消息。
- 系统状态消息。
- Change plan 消息。
- 错误消息。

AI 回答结构：

```text
回答正文

来源
[1] docs/foo.md · 第 12-18 行
[2] specs/bar.md · 标题：索引设计

操作
[复制] [插入] [替换选区] [追加] [新建文档] [生成 diff]
```

视觉规则：

- 用户消息右对齐或弱背景。
- AI 消息左对齐。
- 引用来源用紧凑列表，不做大卡片。
- 操作按钮使用小尺寸 icon + text。
- 流式输出时显示 `停止`。

### 6.5 Composer

结构：

```text
输入框：问当前文档、修改选区或输入 / 选择命令
底栏：[范围] [命令] [模型] [上下文] [估算 3.2k 字符] [发送]
```

交互：

- Enter 发送。
- Shift+Enter 换行。
- `@` 插入文件/标题/附件引用。
- `/` 选择 AI 命令。
- `#` 选择标签上下文。
- `[[` 选择双链文档。

发送前：

- 如果包含工作区/附件且未记住选择，弹上下文确认。
- 如果没有 provider，跳设置。
- 如果上下文超限，打开上下文详情并标记被截断项。

## 7. 上下文预览

### 7.1 预览触发

触发场景：

- 点击发送。
- 运行 AI 命令。
- Agent 任务开始前。
- 自定义命令测试运行。

### 7.2 预览界面

建议用右侧面板内 drawer，不用全屏 modal，避免打断编辑上下文。

结构：

```text
上下文预览
发送到：OpenAI · gpt-5-mini
预计输入：8,420 字符

[ ] 选区
    当前选中 328 字符
[ ] 当前文档
    docs/meeting.md · 标题、目录、正文片段
[ ] 工作区搜索结果
    6 个片段，来自 4 个文件
[ ] 附件
    assets/diagram.pdf · 2 页文本
当前版本不展示 Web 项；如果旧调用传入 Web 请求，只显示“不支持联网搜索”的警告，不访问网络。

[取消] [发送]
```

每个 item 可展开：

- 文件路径。
- 标题。
- 行号。
- 片段预览。
- 字符数。
- 来源类型：全文检索 / 语义检索 / 反链 / 手动选择。

### 7.3 风险提示

风险提示要短而明确：

- `该请求会把工作区片段发送到云端模型。`
- `该请求会读取附件解析文本。`
- `当前版本不支持联网搜索。`
- `当前模型为本地模型，请求会发送到 http://127.0.0.1:11434。`
- `工作区上下文已关闭，未读取其他文件。`

不要使用模糊文案，例如“我们可能会使用一些信息”。

## 8. AI 命令 UI

### 8.1 命令列表

位置：

- 设置 > AI > 命令
- AI 面板内 `命令` drawer

列表列：

```text
启用 | 名称 | 来源 | 范围 | 展示入口 | 默认应用 | 模型 | 操作
```

来源 badge：

- `预置`
- `用户`
- `工作区`
- `插件`

操作：

- 运行
- 编辑
- 复制
- 禁用
- 删除

规则：

- 预置命令不可删除，编辑时生成副本。
- 插件命令不可编辑，只能禁用或查看。
- 工作区命令显示 `.nolia/ai/commands.json` 来源。
- 命令配置损坏时显示错误，但不影响预置命令。

### 8.2 命令编辑器

结构：

```text
命令名称
描述
适用范围：选区 / 段落 / 当前文档 / 文件夹 / 工作区
Prompt 模板
变量插入
默认上下文
默认应用方式
Provider/Model 覆盖
展示入口
测试运行
保存
```

Prompt 编辑器要求：

- 等宽字体。
- 变量高亮。
- 变量 autocomplete。
- 未知变量标红。
- 模板字符数提示。
- 支持恢复上次保存版本。

变量选择：

```text
{{selection}}
{{currentParagraph}}
{{document}}
{{frontmatter}}
{{headings}}
{{workspaceResults}}
{{backlinks}}
{{attachments}}
{{language}}
```

### 8.3 命令测试

测试运行使用当前编辑器 snapshot：

- 如果没有当前文档，提示打开文档。
- 如果命令需要选区但没有选区，允许切换到当前段落或取消。
- 测试结果不自动写入。

## 9. 选区 AI 体验

### 9.1 运行流程

```text
选中文本
-> 选择 AI 命令
-> 显示上下文预览
-> 流式生成
-> 用户选择应用方式
-> 替换/插入进入编辑器 draft
-> 现有保存链路处理落盘
```

### 9.2 结果浮层

短命令可以在编辑器旁显示结果浮层：

```text
润色结果
......

[替换选区] [插入下方] [复制] [在 AI 面板中继续]
```

规则：

- 结果超过 8 行时转入右侧 AI 面板。
- 文件级或工作区级命令直接进入右侧 AI 面板。
- 替换前显示确认，确认文案包含影响范围。

### 9.3 无选区 fallback

无选区时：

- 如果光标在段落中，默认范围为当前段落。
- 如果无法识别段落，默认范围为当前文档。
- UI 必须显示范围，例如 `将处理：当前段落`。

## 10. 工作区问答

### 10.1 范围选择

范围控件：

```text
[当前文档] [当前文件夹] [整个工作区]
```

高级范围：

- 包含反链。
- 包含附件。
- 包含 Web（未来能力，当前版本不展示）。
- 排除当前文件。
- 限定标签。

### 10.2 回答结构

回答必须有三个区块：

1. 直接回答。
2. 来源。
3. 可继续操作。

示例：

```text
结论：这个工作区中，Nolia AI 的核心设计是自研轻量 AI 层...

来源
[1] docs/ai-capabilities/AI_TECHNICAL_DESIGN.md · 第 7-21 行
[2] docs/ai-capabilities/AI_PRODUCT_REQUIREMENTS.md · 第 39-74 行

继续
[打开来源] [插入到当前文档] [生成相关笔记] [推荐双链]
```

### 10.3 来源交互

点击来源：

- 打开对应文件。
- 跳转到行号或标题。
- 高亮片段 2 秒。
- AI 面板保持打开。

来源 hover：

- 显示片段预览。
- 显示来源类型。
- 显示检索分数不对普通用户暴露；调试模式可见。

### 10.4 证据不足

当来源不足：

```text
未在工作区找到明确依据。

可以尝试：
[扩大到整个工作区] [启用语义索引]
```

不能给出无来源的确定结论。

## 11. Agent 变更审核

### 11.1 入口

Agent 类任务从 AI 面板发起：

- 整理当前文档。
- 拆分长文档。
- 为文件夹补标签。
- 生成索引页。
- 批量推荐双链。
- 整理文件夹结构。

### 11.2 Change Plan 总览

布局：

```text
变更计划：整理项目文档
摘要：将创建 1 个索引页，修改 4 个文档，推荐 12 个标签。

操作列表
[ ] 创建 docs/index.md
[ ] 修改 docs/a.md
[ ] 修改 docs/b.md
[ ] 重命名 notes/old.md -> notes/archive/old.md
[ ] 删除 draft/tmp.md

[拒绝全部] [应用已选]
```

删除、移动、重命名使用危险色或警示图标，并默认不勾选。

### 11.3 Diff 审核

修改文件展示：

- 左：原文。
- 右：AI 建议。
- 或统一 diff 视图。

操作：

- 接受整个文件。
- 拒绝整个文件。
- 逐块接受。
- 编辑 AI 建议。
- 打开原文件。

Diff 视图要支持 Markdown：

- 保留行号。
- 新增用绿色边栏。
- 删除用红色边栏。
- 上下文行保持弱色。

### 11.4 应用结果

应用后显示：

```text
已应用 4 项，跳过 1 项。

跳过：
- docs/b.md：保存冲突，磁盘内容已变化。

[查看历史快照] [打开第一个修改文件] [复制报告]
```

## 12. AI 设置体验

### 12.1 Provider 列表

每个 provider item：

```text
OpenAI-compatible
Base URL: https://api.example.com/v1
默认模型：gpt-5-mini
状态：已连接
[设为默认] [测试] [编辑] [删除]
```

本地 provider：

```text
Ollama
http://127.0.0.1:11434
状态：本地服务可用
```

云 provider 必须显示：

```text
内容会发送到第三方模型服务。
```

### 12.2 新增 provider

流程：

1. 选择类型：OpenAI-compatible / Ollama / OpenAI / Anthropic / Gemini / 插件。
2. 填写 label、Base URL、API Key。
3. 测试连接。
4. 选择默认模型。
5. 保存。

API Key 输入：

- 默认 password。
- 支持显示/隐藏。
- 保存后不再回显完整 key。
- 显示 `已保存凭证` 和更新时间。

### 12.3 隐私设置

控件：

- 允许发送当前文档。
- 允许读取工作区搜索片段。
- 允许读取附件解析内容。
- 不提供 Web 搜索开关；当前版本不支持联网搜索。
- 记住上下文确认。
- 保存本地会话历史。

每项必须有说明：

```text
允许读取工作区搜索片段
AI 可以基于全文搜索和语义索引读取其他文件片段。每次请求仍会展示上下文范围。
```

### 12.4 AI 索引设置

展示：

- 状态：关闭 / 待建立 / 索引中 / 已就绪 / 错误。
- 进度。
- chunk 数。
- embedding provider/model。
- 最近更新时间。
- 错误列表。

操作：

- 开启/关闭。
- 重建索引。
- 暂停。
- 清除索引。
- 排除文件夹/标签/扩展名。

索引错误要可读：

```text
无法解析 assets/report.pdf：PDF 文本提取失败。
```

## 13. 空状态、加载和错误

### 13.1 未配置 provider

```text
还没有配置 AI 模型
添加 OpenAI-compatible、Ollama 或其他 provider 后即可使用 AI。

[添加 Provider] [使用本地 Ollama]
```

### 13.2 当前无文档

```text
打开一个 Markdown 文件后，可以总结、改写或提问。

[问整个工作区]
```

### 13.3 无选区

```text
未选择文本，将处理当前段落。
[改为当前文档] [取消]
```

### 13.4 索引未就绪

```text
语义索引还未完成
当前会使用全文搜索和反向链接回答，结果可能不完整。

[查看索引状态] [继续]
```

### 13.5 Provider 错误

错误文案：

- `API Key 无效，请检查 Provider 设置。`
- `模型不存在，请选择其他模型。`
- `网络请求失败，请检查网络或 Base URL。`
- `请求已取消。`
- `上下文过大，请减少范围或切换更大上下文模型。`

错误操作：

- 重试。
- 更换模型。
- 打开设置。
- 缩小上下文。

## 14. 视觉规范

### 14.1 风格

关键词：

- 安静
- 清晰
- 高密度
- 工作台
- 文件可控
- 来源可信

避免：

- 大面积渐变背景。
- 独立 mascot。
- 聊天 App 式全屏泡泡。
- 过度圆角。
- 过多彩色卡片。

### 14.2 颜色语义

建议基于现有主题变量扩展：

```text
--ai-accent: 主题强调色
--ai-local: 本地模型/安全状态
--ai-cloud: 云端模型状态
--ai-warning: 附件/工作区外发提示
--ai-danger: 删除/覆盖/不可逆风险
--ai-citation: 来源引用
```

颜色用途：

- 本地模型：绿色或蓝绿色弱提示。
- 云端模型：中性蓝或紫，但不能让页面变成单一紫色主题。
- 附件/删除：黄色或红色警示。
- 引用：低饱和蓝色链接样式。

### 14.3 字体与密度

- 面板标题：14-15px，600。
- 正文：13-14px。
- 辅助说明：12px。
- 引用、状态、token 估算：11-12px。
- 按钮高度：28-32px。
- 面板内间距：8/12/16px。

不要使用 viewport-based font size。

### 14.4 图标

使用 lucide-react：

- `Sparkles`：AI。
- `Bot`：助手/会话。
- `WandSparkles`：生成/改写。
- `Search`：工作区问答。
- `FileText`：当前文档。
- `Folder`：当前文件夹。
- `DatabaseZap`：语义索引。
- `ShieldCheck`：隐私。
- `KeyRound`：API Key。
- `Globe`：Web，当前版本不使用，保留给未来联网搜索能力。
- `Paperclip`：附件。
- `GitCompare`：diff。
- `Check`：接受。
- `X`：拒绝/关闭。
- `Square`：停止生成。
- `RefreshCcw`：重试/重建。

图标按钮必须有 tooltip 和 aria-label。

## 15. 可访问性

### 15.1 键盘

必须支持：

- Command Palette 调起 AI 命令。
- Tab 遍历 AI 面板控件。
- Escape 关闭 drawer/preview/popover。
- Enter 发送。
- Shift+Enter 换行。
- Ctrl/Cmd+Enter 强制发送或应用主要动作。
- 上下箭头选择命令或来源。

### 15.2 Screen reader

要求：

- 流式输出区域使用 `aria-live="polite"`。
- 错误使用 `role="alert"`。
- 上下文 chip 有明确 label。
- 引用按钮 label 包含文件名和行号。
- diff 行有新增/删除语义文本。

### 15.3 Reduce motion

- 流式输出不做强动效。
- drawer 和 popover 动画可被 `prefers-reduced-motion` 关闭。

## 16. 文案规范

### 16.1 语气

文案要直接、具体、少拟人：

推荐：

- `将发送 6 个工作区片段到 OpenAI。`
- `未在工作区找到明确依据。`
- `替换当前选区前需要确认。`
- `该操作会删除文件，默认未选中。`

避免：

- `我会帮你神奇地整理一切。`
- `放心交给我。`
- `AI 正在思考。`

### 16.2 按钮

主要按钮：

- `发送`
- `停止`
- `替换选区`
- `插入`
- `复制`
- `应用已选`
- `测试连接`
- `重建索引`

危险按钮：

- `删除`
- `应用删除`
- `清除索引`

需要确认的按钮要写清对象：

- `替换当前选区`
- `应用 4 项变更`
- `清除 AI 索引`

## 17. 关键流程

### 17.1 首次使用

```text
点击 AI
-> 显示未配置 provider 空状态
-> 点击添加 Provider
-> 选择 Ollama 或 OpenAI-compatible
-> 测试连接
-> 保存
-> 回到 AI 面板
-> 运行一次当前文档总结
```

成功标准：

- 用户不需要理解技术方案也能完成配置。
- API Key 不回显。
- 本地和云端差异清楚。

### 17.2 选区润色

```text
选中文本
-> 浮动菜单点击润色
-> 上下文预览显示选区
-> 生成结果
-> 用户点击替换选区
-> 确认
-> 编辑器变 dirty
```

成功标准：

- 替换前用户知道影响范围。
- 结果进入编辑器 undo stack。

### 17.3 当前文档问答

```text
打开 AI 面板
-> 范围选择当前文档
-> 输入问题
-> Context Strip 显示当前文档
-> 回答
-> 用户插入到文档
```

成功标准：

- 未保存修改也能作为上下文。
- 回答不自动修改文档。

### 17.4 工作区问答

```text
选择工作区范围
-> 输入问题
-> 上下文预览显示工作区片段
-> 用户确认
-> 回答带来源
-> 点击来源跳转
```

成功标准：

- 来源清楚。
- 无来源时不装作确定。

### 17.5 自定义命令

```text
设置 > AI > 命令
-> 复制“总结为三句话”
-> 修改名称和 Prompt
-> 选择显示在命令面板
-> 保存
-> 命令面板运行
```

成功标准：

- 预置命令不被破坏。
- 自定义命令和预置命令共用上下文确认。

### 17.6 Agent 整理工作区

```text
AI 面板输入整理任务
-> 上下文预览
-> AI 生成变更计划
-> 用户逐项审核
-> 应用已选
-> 冲突项单独反馈
```

成功标准：

- 没有任何文件在审核前被写入。
- 删除/移动默认不选中。

## 20. 不建议做的设计

1. 不做全屏聊天首页。
2. 不让 AI 自动保存或直接写盘。
3. 不把上下文确认藏到设置里。
4. 不只用一个“AI”按钮承载所有能力。
5. 不把来源做成小脚注而不可点击。
6. 不让 provider 配置和命令配置混在一个大表单里。
7. 不在诊断日志或 UI debug 中展示 prompt 正文。
8. 不让插件命令看起来像内置命令，必须标明来源。

## 21. 验收检查清单

### 21.1 基础入口

- AI 面板可以从顶部栏、命令面板和菜单打开。
- 未配置 provider 时显示清晰引导。
- 已配置 provider 时可以直接输入问题。
- 模型和范围在发送前可见。

### 21.2 上下文

- 每次请求前可看到上下文范围。
- 工作区和附件来源有独立标识；当前版本不展示 Web 来源。
- 用户可以取消发送。
- 禁用工作区上下文后，UI 不显示工作区片段。

### 21.3 编辑

- 选区 AI 能读取选区。
- 无选区时显示 fallback 范围。
- 替换选区前确认。
- 插入/替换后文档变 dirty。
- Undo 可以撤销 AI 单点编辑。

### 21.4 来源

- 工作区回答展示来源。
- 来源可点击跳转。
- 无可靠来源时显示证据不足。
- 当前版本不支持 Web 来源；未来支持时必须和本地来源分开。

### 21.5 命令

- 预置命令可运行。
- 预置命令复制后生成自定义命令。
- 自定义命令可编辑、禁用、删除、排序。
- 自定义命令可显示在命令面板。
- 插件命令显示插件来源。

### 21.6 Agent

- Agent 输出变更计划。
- 多文件变更默认不直接应用。
- 删除/移动默认不选中。
- 冲突文件不覆盖。
- 应用后显示成功、跳过和冲突项。

### 21.7 设置

- API Key 不回显。
- 测试连接有明确成功/失败状态。
- 本地模型和云模型标识清楚。
- 索引状态可见。
- 清除索引需要确认。

## 22. 推荐落地顺序

这是设计和实现的依赖顺序，不改变最终产品范围。

1. AI 面板骨架、未配置状态、Provider 设置。
2. Context Strip 和上下文预览。
3. 选区 AI 命令与结果操作。
4. 当前文档问答。
5. 命令管理与自定义命令。
6. 工作区问答与引用来源。
7. AI 索引状态和重建 UI。
8. Agent 变更审核 UI。
9. 附件和插件 provider 的完整提示；Web 搜索作为未来预留能力。

## 23. 最终建议

Nolia AI 的 UI/UX 应坚持“编辑器内生能力”的方向：

- 主体在右侧 AI 面板。
- 高频动作进入命令面板和选区菜单。
- 上下文可见是所有请求的基础。
- 来源引用是工作区问答的核心体验。
- 写入审核是 Agent 的核心体验。
- 设置页必须清楚区分 provider、隐私、索引和命令。

这样的设计可以同时满足普通写作者、知识库用户、技术文档用户和隐私敏感用户，不牺牲 Nolia 已有的本地优先和 Markdown 可控体验。
