# Code of Conduct

Nolia is an open-source project for a local-first desktop writing and knowledge workflow. Project discussions should stay respectful, specific, and focused on improving the software.

## Expected Behavior

- Be professional and considerate in issues, pull requests, reviews, and discussions.
- Critique ideas, code, documentation, and behavior without personal attacks.
- Provide enough context for bug reports, feature requests, and reviews to be actionable.
- Respect maintainers' time and project scope.

## Unacceptable Behavior

- Harassment, threats, discrimination, or personal insults.
- Publishing private information without consent.
- Spam, repeated off-topic comments, or disruptive behavior.
- Attempts to pressure maintainers into unsafe releases or credential disclosure.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; or restrict participation when behavior violates this code of conduct. Security-sensitive reports should follow [SECURITY.md](SECURITY.md).
